# 求职项目知识点清单

## 一、必须掌握（源码实现级别）

### 1. Reactor 网络架构（核心中的核心）

**文件列表**：

- `src/net/eventLoop.h/cpp` - 事件循环核心
- `src/net/channel.h/cpp` - 文件描述符事件封装
- `src/net/epollPoller.h/cpp` - Epoll 多路复用封装
- `src/net/tcpServer.h/cpp` - TCP 服务器
- `src/net/tcpConnection.h/cpp` - TCP 连接管理

**必须掌握的点**：

- One Loop Per Thread 模型的理解
- EventLoop 的事件循环机制（poll -> handleEvent）
- Channel 的职责（封装 fd、注册回调、事件分发）
- EpollPoller 的 Epoll 使用（epoll_ctl、epoll_wait）
- TcpServer 的连接接收流程（Acceptor -> Channel -> 回调）
- TcpConnection 的生命周期管理（建立、读写、关闭）
- 回调机制的设计（使用 std::function 和 std::bind）
- 线程安全的事件唤醒（eventfd 或 pipe）

**面试可能问的问题**：

**1.Reactor 模式是什么？Proactor 与 Reactor 的区别？**

reactor:等就绪 ->用户自己读写

proactor:等就绪->内核读写完了

**2.为什么要用 non-blocking IO？**

单线程高效处理大量连接，必须用非阻塞 IO。

**3.事件循环在做什么？如何保证线程安全？**

- 阻塞在 `epoll_wait` 等 IO 事件
- 将就绪事件分发给 `Channel` 执行回调
- 执行跨线程提交的任务队列 `pendingFunctors`

**Loop 只在固定线程跑，跨线程通过任务队列 + 唤醒机制。**

**4.如何唤醒阻塞在 epoll_wait 的线程？**

往 eventfd 写 8 字节，唤醒阻塞在 epoll_wait 的线程。

**5.连接断开时的处理流程？**

read 返回 0 → 关闭事件 → 状态清理 → 移除连接 → 销毁对象。

### 必须能手写/口述的代码片段

1. **EventLoop 的 loop 函数**
  ```cpp
   void EventLoop::loop()
   {
       while (!m_quit)
       {
           m_activeChannels.clear();
           m_poller->poll(10000, &m_activeChannels);
           for (auto channel : m_activeChannels)
           {
               channel->handleEvent();
           }
       }
   }
  ```
2. **Channel 的 handleEvent 函数**
  ```cpp
   void Channel::handleEvent()
   {
       if (m_revents & EPOLLIN)
       {
           if (m_readCallback) m_readCallback();
       }
       if (m_revents & EPOLLOUT)
       {
           if (m_writeCallback) m_writeCallback();
       }
   }
  ```
3. **Buffer 的 read/write 操作**
  - readerIndex 和 writerIndex 的移动
4. **TcpConnection 的 onMessage 处理**
