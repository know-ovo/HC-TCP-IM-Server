### 一、先看整体状态（3 秒判断方向）

|命令|看什么|异常表现|大概率问题|
|---|---|---|---|
|`htop`|CPU、负载、线程|单核 100%|死循环、计算密集、锁竞争|

### 二、网络端口与连接（服务能不能接入）

|命令|用途|异常|结论|
|---|---|---|---|
|`ss -tulnp`|查看监听端口|无 LISTEN|服务未启动/端口被占/绑定失败|
|`ss -s`|连接总数|ESTAB 很少|客户端没连上/防火墙/backlog 满|
|`ss -tan`|TCP 状态|TIME_WAIT 巨多|短连接风暴，需调内核|
|`ss -s`|连接数|一直不涨|Reactor 线程阻塞|

### 五、典型故障快速对照（直接查表）

#### 1. 服务启动后不接受连接

- ss 无 LISTEN

- 端口被占用 / 权限不足 / bind 失败

- fd 达到上限

- mainReactor 线程阻塞

#### 2. CPU 100%

- 代码死循环

- 大量日志/序列化/加密

- 定时器频繁触发

- 锁自旋/忙等

#### 3. 能连接但不响应

- 日志磁盘 I/O 卡住

- 数据库/同步调用阻塞

- 互斥锁死锁/竞争

- 任务队列堆积

#### 4. 运行一段时间后挂掉

- fd 泄漏 → 达到上限

- 内存泄漏 → OOM kill

- 连接堆积 → 内存爆掉

#### 5. 并发上不去

- 单 Reactor 瓶颈

- 锁竞争严重

- 上下文切换过高

- backlog / 内核参数太小

