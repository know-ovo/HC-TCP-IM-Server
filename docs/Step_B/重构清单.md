# Step_B:将项目从基础骨架升级为工业级可用的Linux高性能IM服务器

## 升级清单：

## P0：最值得优先完成的部分

### 1. 网络层

- 支持非阻塞 I/O
- 支持 LT/ET 两种事件模式，重点实现 ET
- Buffer 升级为环形缓冲区
- 增加发送缓冲区高水位、慢连接治理、背压控制

### 2. 协议与编解码

- 客户端长连接协议从 JSON 升级到 Protobuf
- 协议头补充版本号、请求 ID、序列号、错误码
- 明确协议扩展策略与兼容策略

### 3. 消息可靠性

- ACK 机制
- sequence_id
- 去重窗口
- 重连补拉
- 离线消息持久化
- 消息状态机

### 4. 并发模型

- I/O 线程只负责网络事件
- 业务线程池负责数据库和耗时逻辑
- 通过 `runInLoop()` 回切发送结果

### 5. 数据层

- 引入 MySQL 持久化核心业务数据
- 引入 Redis 管理在线状态和 session

## P1：显著增强项目深度的部分

### 1. 分布式与高可用

- Redis 用户状态中心
- 跨节点会话共享
- 简单的消息路由 / 转发机制
- 节点故障后的重连恢复策略

### 2. 稳定性治理

- 令牌桶限流
- 单接口 QPS 统计
- 熔断与降级
- 心跳超时与空闲连接清理

### 3. 可观测性

- 结构化日志
- Trace ID
- Prometheus 指标
- P99 延迟、在线人数、TPS、失败率监控

### 4. 工程化

- GTest 单测
- clang-format
- clang-tidy
- Google Benchmark
